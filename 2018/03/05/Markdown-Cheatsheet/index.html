This is a personal note about Markdown grammars that I forget so frequently and I REALLY don’t want to Google over and over again.

I collected most of the notes from Markdown-Cheatsheet, Typora Support and answers from StackOverflow and Stack Exchange.

My Markdown engine: Typora.

Emphasis

  1. One * for italics / or one underscore _ will do!

  2. Two *s for bold or two underscores __.

  3. Two ~s for the Strikethrough.

    Please don’t delete me!

    ref: Markdown-Cheatsheet

Lists

  1. First odered list iem
  2. Another item
    • Unodered sub-list
  3. Another item
    1. Use tab to create an ordered sublist
  • Unordered list can use * or - or +
    • use tab to create an un-ordered sublist

ref: Markdown-Cheatsheet

Code and Syntax Highlighting

  1. ` Inline code` —> inline code
  2. Code block & Language

Use ``` surrounding the code block.

Links

Markdown supports two style of links: inline and reference.

[I'm an inline-style link](https://www.google.com) —>

I’m an inline-style link

[I'm an inline-style link with title](https://www.google.com "Google's Homepage") —>

I’m an inline-style link with title

(Not sure what is the difference between the both)

[arbitrary case-insensitive reference text]: any.thing.com "Any title" —>

(you can put all the reference in the end of the file.)

[a relative reference][samefolder/file] —>

[a relative reference][samefolder/file]

Line Breaks

  1. two enters —> new paragragh
  2. two spaces after a line —> new line
  3. One enter will not render new paragraph (at least not in Markdown)
